# Blueprint SDK

The [Blueprint SDK](https://github.com/tangle-network/blueprint/tree/main) is the Rust toolkit for building **blueprints** (off-chain services) and running them as an operator against Tangle’s EVM protocol.

It includes:

- The **[Blueprint Runner](https://github.com/tangle-network/blueprint/tree/main/crates/runner)** (router + producers + consumers + background services)
- Typed **[contexts](https://github.com/tangle-network/blueprint/tree/main/crates/contexts)** (keystore, EVM provider, Tangle EVM client)
- The [`cargo-tangle` CLI](https://github.com/tangle-network/blueprint/tree/main/cli) for creating, preregistering, and running blueprints
- Optional gateways for HTTP ingress and settlement (webhooks, x402)

**Source of truth repos**

- SDK + runner + CLI: https://github.com/tangle-network/blueprint/tree/main
- On-chain contracts: https://github.com/tangle-network/tnt-core/tree/main

## What You Run (Operator Tooling)

- [`cargo-tangle`](https://github.com/tangle-network/blueprint/tree/main/cli): developer/operator CLI (register, run, service lifecycle, jobs)
- [`blueprint-manager`](https://github.com/tangle-network/blueprint/tree/main/crates/manager): long-lived operator runtime that watches chain events and runs services
- [`pricing-engine-server`](https://github.com/tangle-network/blueprint/tree/main/crates/pricing-engine): optional RFQ server that issues signed quotes

## Configuration Surface (Tangle EVM)

Most tooling needs the same deployment addresses and endpoints:

- `http_rpc_url` / `ws_rpc_url`
- `tangle_contract` (core protocol)
- `staking_contract` (MultiAssetDelegation)
- `status_registry_contract` (heartbeats; optional but recommended)
- `keystore_path`
